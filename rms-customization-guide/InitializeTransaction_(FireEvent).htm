InitializeTransaction Event

The handler for the InitializeTransaction event initializes a new transaction.

Handler Syntax

Public Function InitializeTransaction( _

) As Boolean

Return Values

Boolean value that indicates whether the operation was a success. The event handler returns TRUE if the operation was a success; otherwise, FALSE.

UI Equivalent Key

ESC when a transaction is in progress

Remarks

The handler for the InitializeTransaction event does not prompt the user to confirm the initialization of the new transaction. If a transaction is in progress when this event is raised the current transaction will be lost.

If the Manager application is configured to require anything for transactions, for example that a customer or sales representative be loaded, the POS prompts the user for the required information.

The handler for this event performs the following actions.

    1. Removes any items currently held on reserve for this register.

    2. Resets all QSRules objects associated with the transaction; for example, the Transaction object and all TransactionEntry and Tender objects are reset.

    3. Sets the state of the transaction to the preset defaults; for example, if default shipping information exists, that information will be set to the transaction.

    4. Clears the transaction grid.

    5. Prompts the user for any transaction information required by the Manager configuration.
