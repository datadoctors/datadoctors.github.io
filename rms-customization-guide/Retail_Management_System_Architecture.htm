Retail Management System Architecture

This section provides an architectural overview of Store Operations and Headquarters, with details about the architectural components used by these applications.

Store Operations

Critical architectural components deployed at the store in support of the EXE programs include the following:

  • QSRules — Encapsulates business logic and database communication for the POS and Manager executable programs. QSRules is the heart of Store Operations. When you develop add-ins for Store Operations, you are building on top of QSRules. The public properties in QSRules and the methods used by add-in programs provide the ability to create powerful custom solutions.

  • Store Operations Database — Contains the data that drives Store Operations, using Microsoft SQL Server or MSDE.

  • Headquarters Client — Deployed at each store to communicate with Headquarters Server and update the Store Operations database.

 

Headquarters

Critical architectural components deployed at the head office in support of the EXE programs include the following:

  • Headquarters Server Rules — An in-process ActiveX DLL that encapsulates business logic and database connection for Headquarters Server.

  • Headquarters Rules — An in-process ActiveX DLL that encapsulates business logic and database connection for Headquarters Manager.

  • Headquarters Database — Contains the data for Headquarters, using Microsoft SQL Server or MSDE.
