SUB Element

The SUB element defines a subprocedure that can be invoked by a CALL element.

Attributes

Attribute

Value

Description

name

String

Required. String used to identify the subprocedure in a CALL element.

Value

None

Child Elements

Any

Remarks

SUB elements are used like a subprocedure in Visual Basic or VBScript. The content of a SUB element is evaluated when the element is invoked by a CALL element. SUB elements are typically used to create duplicate blocks of information in a specific location on specific pages of a receipt with more than one page.

The following SUB elements are called automatically by the receipt renderer if they exist in a template, and are used primarily for headers, footers, and summary information.

  • <sub name="OverlayFirstPage">
    Invoked on the first page if the receipt contains two or more pages.

  • <sub name="OverlayMiddlePage">
    Invoked on any page after the first page and before the last page, if the receipt contains three or more pages.

  • <sub name="OverlayLastPage">
    Invoked on the last page if the receipt contains two or more pages.

  • <sub name="OverlayOnlyPage">
    Invoked if the receipt contains only one page.

Example

The following example shows how to define and call a subprocedure.

<SUB name="MySub">

    <PARAGRAPH>

        <TEXT> "Entering MySub." </TEXT>

    </PARAGRAPH>

</SUB>

 

<DOCUMENT>

    <CALL> "MySub" </CALL>

</DOCUMENT>
