ReportSummary Report Template

The Report Summary defines the basic source, grouping, and sorting information for the report. It contains the following properties:

 

Property

Description

ReportType

  • reporttypeCustomer

  • reporttypeSales

  • reporttypeItems

  • reporttypeOrders

  • reporttypeLayaway

  • reporttypeMisc

  • reporttypeCustom

  • reportypeMemorized

ReportTitle

String

Title of report

PageOrientation

  • pageorientationPortrait

  • pageorientationLandscape

Sets the default size of the report window in Manager. Use Landscape for reports with many columns.

OutlineMode

  • True

  • False

Determines if the "tree" outline structure is used when grouping.

Groups

Number

Specifies the number of levels to automatically group the report data.

GroupDescription

String

Title for summary rows in outline-mode.

DisplayLogo

  • True

  • False

LogoFileName

String

Name of logo file name in the Retail Management Systems picture folder.

ProcedureCall

String

Name of SQL StoredProcedure to run. Enter "" if not used.

PreQuery1

String

A series of valid SQL statements to be performed before the main report query. If more than one statement is included, they must be separated by the string "[GO]". For example:

"Drop Table #Temp1 [Go] Drop Table #Temp2"

Note:When a report is generated, the sequence of execution for the various SQL statements is as follows:

  1. PreQuery1

  2. PreQuery2

  3. Main report query based on TablesQueried, SelCriteria, visible Columns, and Filters.

PreQuery2

String

A series of valid SQL statements to be performed before the main report query, but after PreQuery1.

For more details, see PreQuery1 above.

TablesQueried

String

A valid SQL FROM clause. For example:

"FROM Item LEFT JOIN Department ON Item.DepartmentID = Department.ID"

To use a statement that spans multiple lines, do NOT enclose the statement in quotation marks. Instead prefix the statement with "<BEGIN>" and terminate it with "<END>".

For example:

TablesQueried =  

<BEGIN>
   FROM Item
   LEFT JOIN Department ON Item.DepartmentID =  DepartmentID
<END>

SelCriteria

String

An expression suitable for inclusion in a SQL WHERE clause, such as:

"(Item.SalePrice > 0.0)"

Note: To ensure that the selection criteria is properly evaluated when filters are used, surround it with parentheses.

GroupBy

String

Column name to group results by, per SQL GROUP BY clause, such as:

"Cashier.Number"

SortOrder

String

Column name to sort by, per SQL ORDER BY clause. May end with "ASC" or "DESC" to specify direction of sort order, such as:

"Cashier.Name DESC"

 

The following variables are valid in the report summary:

Variable

Description

<MonthStart>

Date of the first day of the month.

<WeekStart>

Date of the first day of this week.

<LastWeekStart>

Date of the first day of last week.

<YearStart>

Date of the first day of the year.

<Now>

Current date and time.

<Report Date>

Date of report.

<Today>

Today's date.

<Report Title>

The title of the report.

<Store Name>

Name of the store.

<Store Address1>

First line of the address of the store.

<Store Address2>

Second line of the address of the store.

<StoreCity>

City of the address of the store.

<StoreState>

State code of the address of the store.

<Store Zip>

Postal code of the address of the store.

<Store Country>

Country code of the address of the store.

<Store Phone>

Telephone number of the store.

<Store Fax>

Facsimile number of the store.
