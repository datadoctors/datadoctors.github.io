QSRules Classes

COM component and HTML script add-ins use QSRules classes to access the business logic tier of Store Operations POS.

The following table lists the documented QSRules classes.

Class

Description

Cashier

Represents a single cashier.

Configuration

Provides access to the configuration set in the RMS Manager application.

CurrencyDescriptor

Contains static information about a currency.

Customer

Represents a single customer.

Post

Contains information about the posting of a batch to the integrated accounting software.

PostAccount

Represents a single account in the accounting software integrated with RMS.

PostEntry

Represents a single entry in the batch being posted to the accounting software integrated with RMS.

ReasonCode

Represents a reason code for a discount, return, or tax change.

Register

Represents a single POS register.

SessionClass

Primary entry point for DLL and HTML add-ins.

Shipping

Contains shipping information for a transaction.

Status

Contains information about the current status.

Tender

Represents a single tender type.

TenderDescriptor

Contains static information about a tender.

Transaction

Represents a POS transaction of any type, including sales, work orders, back orders, quotes, and layaways.

TransactionEntry

Represents a single line entry in the transaction.
