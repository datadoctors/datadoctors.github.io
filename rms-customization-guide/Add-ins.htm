This example shows the series of events for a COM component add-in invoked by the AddItem hook in POS.

    1. The cashier scans an item.

    2. The POS reads the item lookup code and instructs QSRules to add the transaction entry.

    3. QSRules creates a new TransactionEntry object in the TransactionEntries collection contained in the Transaction object and populates it with data from the scanned item.

    4. The POS reads the registry and detects that an add-in is to be invoked from the AddItem hook.

    5. The POS performs a CreateObject operation to create the COM component add-in object defined in the registry and invokes the Process function in the class specified in the registry, passing it a pointer to the head SessionClass object.

    6. The add-in interacts with QSRules to read information and optionally modify the transaction.

    7. The add-in completes processing and control returns to the POS.

 

Add-ins

There are three types of add-ins for Store Operations:

  • COM component (for Manager and POS)

  • Standard EXE (for Manager)

  • HTML script (for POS)

COM Component Add-ins

You can use COM component add-ins in either Store Operations Manager or POS.

A COM component add-in can interact with Store Operations in one of two ways:

  • COM component add-ins can "hook" into a specific point in Manager or POS. This is accomplished from a registry entry that describes the add-in. The add-in is passed either a reference to the head SessionClass object in QSRules or the ID of a record just stored in the database.

Example

  • A custom POS button can trigger a COM component add-in. The add-in is passed a reference to the Session class in QSRules (Custom POS buttons are defined in Manager and appear on the right side of the POS screen.)

For more details, see COM Component Add-ins.

Standard EXE Add-ins

You can use standard EXE add-ins in Store Operations Manager. These add-ins are triggered from a sub-menu of the Add-Ins command on the Utilities menu.

Store Operations automatically displays these add-ins on the sub-menu when you store the add-in in the C:\Program Files\Microsoft Retail Management System\Store Operations\AddIns folder. You can store these add-ins in a different folder as long as you change the path in Store Operations Administrator. (Click Configuration on the File menu, and then click the Paths tab.)

Store Operations stores the add-in path in the
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Retail Management System\Store Operations\Manager\Path\AddIns subkey.

For more details, see Standard EXE Add-ins.

HTML Script Add-ins

You can use HTML script add-ins in Store Operations POS. These add-ins can greatly extend the functionality and the look of the POS. The POS screen includes three embedded browsers:

  • HTML Status Bar — Located at the top of the POS screen. This browser is a good place to display status information about the current transaction. It can also display a non-interactive Web page.

  • HTML Browser — Replaces the transaction grid on the POS screen when invoked by a custom POS button. You can program this browser to interact with the user, as well as display status information and Web pages.

  • Net Display — Runs on a secondary customer-facing monitor and consists of two HTML panes: a receipt pane and an advertising pane. The receipt pane displays a running receipt of the current transaction. The advertising pane displays advertisements or Web pages that can be triggered by events during a sales transaction.

All of the browsers communicate with the POS program through a COM component called QSBridge. In addition to accessing the business logic in QSRules, these browsers can fire events in the user interface layer of the POS.

For more details, see HTML Add-ins.
