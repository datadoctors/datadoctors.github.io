PerformAddItem Event

The handler for the PerformAddItem event adds an new item to the POS screen and gathers any required data, such as a serial number, matrix information, and error messages.

Handler Syntax

Public Function PerformAddItem( _

  ItemID As Long, _

  ItemLookupCode As String, _

  Quantity As Double, _

  OverridePrice As Boolean, _

  Price As Currency, _

  OverrideDescription As Boolean, _

  Description As String

) As Boolean

Parameters

ItemID

Long value that contains the ID in the database of the item to add to the transaction. If you do not know the ID of the item in the database, set this value to zero and use the ItemLookupCode parameter to identify the item. This parameter overrides the ItemLookupCode parameter.

ItemLookupCode

String that contains the Item Lookup Code of the item to add. This parameter is not used if the ItemID parameter is not zero.

Quantity

Double value that contains the quantity of the item to add to the transaction. If this value is zero an entry for the item is added to the transaction with a quantity of zero; if the quantity of the item is zero when the transaction is tendered, the item will not be tendered. If this value is less than zero the item entry is treated as an item return.

OverridePrice

Boolean value that specifies whether to override the price for the item.

Price

Currency value that contains the new price to set to the item. This parameter is only used if the OverridePrice parameter is passed TRUE.

OverrideDescription

Boolean value that specifies whether to override the description of the item.

Description

String that contains the new description for the item. This parameter is only used if the OverrideDescription parameter is passed TRUE.

Return Values

Boolean value that indicates whether the operation was a success. The event handler returns TRUE if the operation was a success; otherwise, FALSE.

UI Equivalent Key

  • F2 and select an item from the Items dialog

  • Item Lookup Code typed in the Lookup Code column 

Remarks

If an exact match is not found in the database the handler for the PerformAddItem event searches for a matching composite item; if the item is identified as a Matrix, Lot Matrix, or Assembly item the handler processes the item class according to its type.

If an item is found, the handler for this event performs the following actions.

    1. Deletes the current transaction item if it is not blank.

    2. Adds a new item, including tag-along items if applicable, to SessionClass.Transaction.Entries.

    3. Updates the transaction grid with item descriptions for the item and tag-along items added.

    4. Recalculates totals, taxes, purchase limits, and discounts.

    5. Reads the weight from the scale if the type is a weighed item.

    6. Negates the quantity if the transaction is in return mode.

    7. Updates the quantity for the entry.

      • If the transaction is a sales transaction, sets TransactionEntry.QuantityPurchased equal to the specified quantity.

      • If the transaction is not a sales transaction, sets TransactionEntry.QuantityOnOrder equal to the specified quantity.

    8. Validates the quantity against the quantity available.

    9. Prompts for the serial number, if applicable.

    10. Prompts for price entry, if applicable.

    11. Overrides the price and description for the item if the parameters are provided.
