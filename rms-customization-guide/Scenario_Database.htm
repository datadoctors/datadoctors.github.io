Loyalty Add-in Database

The loyalty database contains all information about the loyalty program. This database is created during installation on the same server used by RMS. Creating a separate database allows new information to be stored without modifying the schema of the RMS database.

The loyalty database contains the following tables:

  • LoyaltyLevel table: Stores information about the defined loyalty levels. For more information, see LoyaltyLevel Table in this topic.

  • LoyaltySignup table: Stores loyalty information for the customers enrolled in the loyalty program. For more information, see LoyaltySignup Table in this topic.

  • Message table: Stores the messages used by the loyalty program. For more information, see Message Table in this topic.

  • ErrorLog table: Stores information about all loyalty add-in errors. For more information, see ErrorLog Table in this topic.

 

LoyaltyLevel Table

The LoyaltyLevel table contains information about the loyalty levels defined by the user. This information is set by the user during installation, and can be modified in Manager. The fields in the LoyaltyLevel table are the following.

Field

Type

Description

LoyaltyLevelID

int

Key field. This field is automatically generated.

LoyaltyLevelName

nvarchar

Name of the loyalty level, for example "Gold".

Amount

money

Purchase amount required to reach the loyalty level.

DiscountPercentage

real

Discount percentage applied to purchases made by customers at the loyalty level.

 

LoyaltySignup Table

The LoyaltySignup table contains loyalty information for the customers enrolled in the loyalty program. The fields in the LoyaltySignup table are the following.

Field

Type

Description

CustomerID

int

Key field. This field is automatically generated.

LoyaltyJoinedDate

datetime

Date on which the customer enrolled in the loyalty program.

ReceiveMailings

bit

Boolean value that indicates whether the customer receives store mailings.

SignedUpCashierID

int

ID of the entry in the RMS Cashier table for the cashier that signed the customer up for the loyalty program.

LoyaltySales

money

Current loyalty program purchase amount for the customer.

LastUpdated

datetime

Date on which the customer loyalty enrollment was last updated.

LoyaltyLevelID

int

ID of the entry in the LoyaltyLevel table for the current loyalty level of the customer.

 

Message Table

The Message table contains the loyalty program messages. This information is set by the user during installation, and can be modified in Manager. The fields in the Message table are the following.

Field

Type

Description

ID

nvarchar

Key field that identifies the message. These values are set by the loyalty installation, and should not be changed.

MessageText

nvarchar

Text of the message.

LastUpdated

datetime

Date on which the message was last updated.

 

ErrorLog Table

The ErrorLog table contains information about all errors that occur in the loyalty add-in. A new row is added to the table whenever an error occurs. The fields in the ErrorLog table are the following.

Field

Type

Description

ErrCode

varchar

ID of the error.

ErrDescription

varchar

Human-readable description of the error.

UpdateDate

smalldatetime

Date on which the error occurred.
