  7. Strongname your assembly by typing the following command at the prompt.

  1. Open the assembly file (AssemblyInfo.vb) in your project folder and add the following line.

  1. Save the assembly file and recompile your project.

  2. At the command prompt, navigate to the bin folder of your project.

  3. Create and register a type library for your DLL by typing the following command at the prompt.

The Process() Function

The entry point for most DLLs is the Process function. This function has the following signature.

Visual Basic 6.0

Public Function Process(mySession As QSRules.SessionClass) As Boolean

Visual Basic .NET

Public Function Process(mySession As Object) As Boolean

Because QSRules.dll is not strong named, you cannot directly reference it in Visual Basic .NET. Instead, you must declare the reference to the main SessionClass object as the generic type Object. As mentioned in the previous section, it is useful in Visual Basic 6.0 to use the SessionClass object while creating and testing the component, but the signature should be changed to use the generic Object class before deployment.

When a COM component add-in is called through a hook or custom POS button, RMS passes a reference to the current session to the mySession parameter. Use this reference to a SessionClass object to interact with the RMS business logic.

The return value of the Process function is used by RMS to determine whether the operation was a success.

To see how the loyalty add-in uses the Process function, see Loyalty Level Computation Sample.

The following simple example shows how to use the reference to the SessionClass object to access the name of the cashier currently logged on to the POS.

Visual Basic .NET, Visual Basic 6.0

Public Function Process(mySession As Object) As Boolean

    If MsgBox("The current cashier is " _

               & mySession.Cashier.Name & ".") Then

        Process = True

    Else

        Process = False

    End If

End Function

 

Accounting Functions

The ExportAccounting hook is the only hook that does not use the Process function as an entry point. Instead, this hook uses the following functions.

  • BeginPost function: This function is called once before any accounting information is posted, and has the following signature.

Public Function BeginPost (mySession As Object) As Boolean

  • Post function: This function is called once before each accounting record is posted, and has the following signature.

Public Function Post (mySession As Object) As Boolean

  • EndPost function: This function is called once after all the accounting information is posted, and has the following signature.

Public Function EndPost (mySession As Object) As Boolean

For more information about posting batches to accounting software, see the ExportAccounting hook and the Post class.
