Tender Class

The Tender class represents a single tender type.

 

When To Use

The Tender class is used to perform the following tasks.

  • Retrieve information about a tender type.

  • Retrieve amounts paid out and taken in for a tender in the default currency type for the tender as well as the local currency.

  • Load, validate, and store credit card, debit card, and check information.

 

Procedures

The Tender class defines the following procedures.

Method

Description

ExtractCreditCardNumber

Extracts the credit card information from the specified magnetically swiped credit card data.

Reset

Resets all of the properties for this class to their default values.

ResetValidatedVoucher

Resets the validation for the voucher to an invalidated state.

SetPrinterValidationOccurred

Marks the tender as validated by the printer.

ValidateAmount

Validates the tender amount against the maximum amount defined for the tender and ensures that the tender is not overtendered.

ValidateCheckInfo

Saves the specified check information.

ValidateCheckPositiveNegative

Performs positive/negative check validation on the check.

ValidateCreditCardInfo

Validates and saves the specified credit card information.

ValidateDebitCardInfo

Validates and saves the specified debit card information.

ValidateSignature

Saves the specified signature.

 

Properties

The Tender class has the following properties.

 

AccountExpiration

Data type: String

Access type: Read/Write

Gets or sets the expiration date for credit cards or debit cards.

 

AccountHolder

Data type: String

Access type: Read/Write

Gets or sets the name of the person that holds the account for the tender.

 

AccountNumber

Data type: String

Access type: Read/Write

Gets or sets the check, credit card, or debit card account number.

 

AccountNumberMasked

Data type: String

Access type: Read-only

Gets the masked account number for checks, credit cards, or debit cards.

 

AdditionalSecurityInfo

Data type: String

Access type: Read/Write

Gets or sets additional security information for debit cards.

 

AmountIn

Data type: Currency

Access type: Read/Write

Gets or sets the total amount of the tender taken in.

 

AmountInLocal

Data type: Currency

Access type: Read-only

Gets the total amount of the tender taken in, in the local currency.

 

AmountInLocalRounding

Data type: Currency

Access type: Read-only

Gets the amount of the local currency left over when the amount of the tender taken in is converted to its equivalent in local currency.

 

AmountOut

Data type: Currency

Access type: Read/Write

Gets or sets the total amount of the tender paid out.

 

AmountOutLocal

Data type: Currency

Access type: Read-only

Gets the total amount of the tender paid out, in the local currency.

 

AmountOutLocalRounding

Data type: Currency

Access type: Read-only

Gets the amount of the local currency left over when the amount of the tender paid out is converted to its equivalent in local currency.

 

ApprovalCode

Data type: String

Access type: Read-only

Gets the approval code for credit cards and debit cards.

 

BirthDate

Data type: String

Access type: Read-only

Gets the birth date for the signer of the check.

 

CashBackAmount

Data type: Currency

Access type: Read/Write

Gets the amount of the tender returned as cash back for debit cards.

 

CashBackSurcharge

Data type: Currency

Access type: Read-only

Gets the surcharge applied when a debit card is used to get cash back.

 

DebitAllSurcharges

Data type: Currency

Access type: Read-only

Gets the total surcharge applied for debit cards.

 

DebitSurcharge

Data type: Currency

Access type: Read-only

Gets the surcharge applied for debit cards.

 

Descriptor

Data type: TenderDescriptor

Access type: Read-only

Gets the static information for the tender.

 

Display

Data type: Boolean

Access type: Read/Write

Gets or sets a Boolean value that determines whether the tender is displayed on the Tender dialog box.

 

DriversLicense

Data type: String

Access type: Read-only

Gets the driver license number for checks.

 

EncryptedPIN

Data type: String

Access type: Read/Write

Gets or sets the encrypted PIN for debit cards.

 

Entered

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether an amount has been entered for the tender.

 

HasVoucherExpirationDate

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the tender is a voucher with an expiration date.

 

Key

Data type: String

Access type: Read-only

Gets the key of the tender.

 

MICRRawData

Data type: String

Access type: Read-only

Gets the raw data retrieved from a check using Magnetic Ink Character Recognition (MICR).

 

RequireAmountValidation

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the amount for the tender needs to be validated.

 

RequireCheckInfo

Data type: Boolean

Access type: Read-only

RequireCheckPositiveNegativeValidation

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the check requires positive/negative validation.

 

RequireCreditCardInfo

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether credit card information needs to be loaded.

 

RequireDebitCardInfo

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether debit card information needs to loaded.

 

RequirePrinterValidation

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether printer validation is required.

 

RequireSignatureValidation

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether signature validation is required.

 

SerialNumber

Data type: String

Access type: Read-only

Gets the serial number of the check.

 

Signature

Data type: String

Access type: Read-only

Gets the signature for the tender.

 

StateCode

Data type: String

Access type: Read-only

Gets the code for the state for the check.

 

Swiped

Data type: Boolean

Access type: Read-only

Track1

Data type: String

Access type: Read-only

Track2

Data type: String

Access type: Read-only

Gets the information contained on the second track of the credit card or debit card.

 

TransitNumber

Data type: String

Access type: Read-only

Gets the transit number of the check.

 

VoucherNumber

Data type: String

Access type: Read-only
