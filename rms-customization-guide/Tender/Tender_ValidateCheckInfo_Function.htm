Tender.ValidateCheckInfo Function

The ValidateCheckInfo function saves the specified check information.

Public Function ValidateCheckInfo( _

  ByVal AccountNumber As String, _

  ByVal DriverLicense As String, _

  ByVal StateCode As String, _

  ByVal SerialNumber As String, _

  ByVal TransitNumber As String, _

  ByVal MICRRawData As String, _

  ByVal BirthDate As String _

) As StatusCategoryEnum

Parameters

AccountNumber

String that contains the account number for the check. This value is stored in the AccountNumber property.

DriverLicense

String that contains the driver license number for the check. This value is stored in the DriversLicense property.

StateCode

String that contains the two-character state code for the check. This value is stored in the StateCode property.

SerialNumber

String that contains the serial number for the check. This value is stored in the SerialNumber property.

TransitNumber

String that contains the transit number for the check. This value is stored in the TransitNumber property.

MICRRawData

String that contains the Magnetic Ink Character Recognition information for the check. This value is stored in the MICRRawData property.

BirthDate

String that contains the birth date for the signer of the check. This value is stored in the BirthDate property.

Return Values

StatusCategoryEnum value that indicates whether the operation was a success.

Remarks

You should only call this function when the RequireCheckInfo property is TRUE. The specified information is stored in the TenderEntry table after posting but is not validated.
