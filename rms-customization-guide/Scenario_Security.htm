Loyalty Add-in Security

The loyalty database contains sensitive information about customers. This information should only be accessed through the POS and Manager applications, which require users to log on. Although the loyalty database requires a user ID and password, the connection string used to access the database from the loyalty solution contains this logon information in plain text.

To prevent unauthorized users from retrieving the database logon information from the connection string, the loyalty solution encrypts the connection string and stores the encrypted form in the registry. All connections to the loyalty database are made through a dedicated class that decrypts the connection string, establishes the connection, and passes the connection to the class that needs to access the database.

The encryption method used depends on the operating system.

 

Windows XP and Windows 2000

In Windows XP and Windows 2000, the loyalty solution uses the Win32 Data Protection API (DPAPI) to encrypt and decrypt the connection string. DPAPI derives the encryption key from the password of the user account associated with the code that calls the DPAPI functions. As a result, the operating system, not the application, manages the key. DPAPI can work with either the machine store or the user store; the loyalty application uses the machine store. This means that different encrypted data is generated on every computer.

DPAPI is accessed through Crypt32.dll. The following example shows how to import the DPAPI cryptographic functions from Crypt32.dll.

Private Declare Function CryptProtectData Lib "crypt32.dll" ( _

    ByRef pDataIn As DATA_BLOB, ByVal szDataDescr As String, _

    ByRef pOptionalEntropy As DATA_BLOB, ByVal pvReserved As IntPtr, _

    ByVal pPromptStruct As IntPtr, ByVal dwFlags As Integer, _

    ByRef pDataOut As DATA_BLOB _

) As Boolean

 

Private Declare Function CryptUnprotectData Lib "crypt32.dll" ( _

    ByRef pDataIn As DATA_BLOB, ByVal szDataDescr As String, _

    ByRef pOptionalEntropy As DATA_BLOB, ByVal pvReserved As IntPtr, _

    ByVal pPromptStruct As IntPtr, ByVal dwFlags As Integer, _

    ByRef pDataOut As DATA_BLOB _

) As Boolean

 

The following example shows how the data is encrypted using DPAPI.

CryptProtectData( _

    plainTextBlob, _                ' Data to encrypt.

    "LoyaltyCrypt", _               ' Description of the encryption.

    Nothing, Nothing, Nothing, _

    CRYPTPROTECT_LOCAL_MACHINE Or CRYPTPROTECT_UI_FORBIDDEN, _

    cipherTextBlob _                ' Returns the encrypted data.

)

 

The following example shows how the data is decrypted using DPAPI.

CryptUnprotectData( _

    cipherBlob, _                   ' Data to decrypt.

    "LoyaltyCrypt", _               ' Description of the encryption.

    Nothing, Nothing, Nothing, _    

    CRYPTPROTECT_LOCAL_MACHINE Or CRYPTPROTECT_UI_FORBIDDEN, _

    plainTextBlob _                 ' Returns the decrypted data.

)

 

These examples use the following flags:

  • CRYPTPROTECT_LOCAL_MACHINE: Associates the encrypted data with the current computer instead of with an individual user. Any user on the computer on which the CryptProtectData function is called can use the CryptUnprotectData function to decrypt the data.

  • CRYPTPROTECT_UI_FORBIDDEN: Prevents remote computers from decrypting the data when the UI cannot be protected.

 

Windows 98

In Windows 98, the loyalty solution uses Crypto API to encrypt and decrypt the connection string. The connection string is encrypted using the Rijndael algorithm, and the encrypted string is stored in the registry as bytes.

The following example shows how the data is encrypted using Crypto API. The key and initialization vector are declared globally.

' Set the key and initialization vector.

myRijndael.Key = key

myRijndael.IV = IV

 

' Get an encryptor.

Dim encryptor As ICryptoTransform = myRijndael.CreateEncryptor(key, IV)

 

' Encrypt the data.

Dim msEncrypt As New MemoryStream

Dim csEncrypt As New CryptoStream(msEncrypt, encryptor, CryptoStreamMode.Write)

 

' Convert the data to a byte array.

toEncrypt = textConverter.GetBytes(dataString)

 

' Write all data to the crypto stream and flush it.

csEncrypt.Write(toEncrypt, 0, toEncrypt.Length)

csEncrypt.FlushFinalBlock()

 

' Get the encrypted array of bytes.

Encrypt98 = msEncrypt.ToArray()

 

The following example shows how the data is decrypted using Crypto API.

' Set the key and initialization vector.

myRijndael.Key = key

myRijndael.IV = IV

 

' Get a decryptor that uses the same key and IV as the encryptor.

Dim decryptor As ICryptoTransform = myRijndael.CreateDecryptor(key, IV)

 

' Decrypt the previously encrypted message using the decryptor.

Dim msDecrypt As New MemoryStream(encrypted)

Dim csDecrypt As New CryptoStream(msDecrypt, decryptor, CryptoStreamMode.Read)

 

' Holds the data to be encrypted in bytes.

fromEncrypt = New Byte(encrypted.Length) {}

 

' Read the data from the crypto stream.

csDecrypt.Read(fromEncrypt, 0, fromEncrypt.Length)

 

' Convert the byte array back into a string.

Decrypt98 = textConverter.GetString(fromEncrypt)
