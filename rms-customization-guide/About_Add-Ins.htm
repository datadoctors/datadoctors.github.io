About Add-ins

You can extend the functionality of Microsoft® Retail Management System by creating custom add-ins that interact with the Store Operations business logic. There are three types of add-ins, each of which extends the functionality of RMS in different ways.

  • COM component add-ins are the most flexible type of add-in. These add-ins interact with the transaction in progress using QSRules, and can be triggered at specific points in the transaction flow or activated from a custom POS button.

  • HTML add-ins display information about the transaction in one of the HTML browsers embedded in the POS. In addition, HTML pages displayed in the internal HTML browser can accept information from the user.

  • Standard EXE add-ins provide external utilities to POS or Manager. These add-ins are not passed a reference to QSRules, and thus cannot interact directly with Store Operations.

Add-in Best Practices

Use the following guidelines to produce high-quality add-ins.

  • When running your add-in in an environment with other add-ins:

  • Use the first empty registry entry when installing your hook and increment the hook count for your add-ins. For more details, see Invoking COM Components from Hooks.

  • Find the first empty variable and use it for your add-in. Do not assume that your add-in has complete access to the generic variables in the SessionClass class. For more information, see the SessionClass class.

  • Modify the properties in QSRules classes wherever possible; let QSRules post the transaction.

  • Do not modify existing tables in the database.

  • Future versions of Microsoft Retail Management System will be in .NET. You can create add-ins for the current version either by writing in Visual Studio 6 or by writing in .NET with a COM wrapper. If you write in Visual Studio 6, you will need to port the add-ins to .NET for future versions of Retail Management System. For more information about creating a COM add-in in Visual Basic .NET, see Creating a COM Component Add-in.
