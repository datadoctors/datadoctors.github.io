Customization

You can customize Microsoft® Retail Management System in numerous ways. Whether you want to extend functionality by customizing the user interface or simply intercept a transaction to provide further processing, Retail Management System’s open architecture provides developers with the ability to deliver specialized solutions. Customizations fall into three categories:

POS Customizations

  • Provide additional sales transaction processing by creating add-ins that interact with a transaction's business logic.

  • Modify the POS screen in several ways:

  • Add custom buttons on the right side of the screen.

  • Add a custom HTML status bar at the top of the screen.

  • Replace the standard sales transaction screen with a custom browser.

  • Add a customer-facing monitor with a running receipt and/or linked Web pages.

  • Create custom graphical receipts and reports that use both standard information and information from your own add-ins.

Manager Customizations

  • Provide additional functionality by creating add-ins that are triggered by menu commands or other events in Manager.

  • Create custom reports, purchase order templates, and account statements.

Integration

  • Provides integration with other software packages.

  • Provides integration with payment providers.

  • Provides integration with Yahoo Stores. Orders received at Yahoo Stores can be processed directly as sales transactions in Store Operations.
