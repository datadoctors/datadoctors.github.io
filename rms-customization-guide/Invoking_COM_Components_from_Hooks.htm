Invoking COM Components from Hooks

Retail Management System gathers data about external hook add-ins from the registry. Once an add-in has been added to the registry the add-in will be called whenever the specified hook is fired in Store Operations; for example, if you set up a DLL to be called by the AddItem hook, the DLL will be called every time an item is added to the transaction.

To add an add-in to the registry:

  1. Ensure that the add-in is registered on the local computer.

  2. Increment the value of the Count entry of the \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Retail Management System\Store Operations\Hooks subkey by one.

  3. Edit the values of the \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Retail Management System\Store Operations\Hooks\Number subkey entries (where Number is the number of the first unused hook, between 000 and 029) with the following required information.

      • HookType: Enter the number of the hook type that will activate the add-in. For more information, see Hook Types in this topic.

      • ObjectName: Enter the name of your DLL and the name of the class that contains the entry point function, separated by a period. For example, if the name of your DLL is MyDLL and the name of the class that contains the Process function is MyClass, the object name is "MyDLL.MyClass".

      • Parameter: Enter the number of the parameter required by the specified hook type. The available parameters are listed in the following table.

  1. Add the following descriptive information to the subkey. These entries are not required, but help to identify the DLL.

      • Caption: Enter identifying text for the add-in. For hooks of type ExportAccounting, this caption is added to the list of available accounting software on the Accounting tab in the Store Operations Manager application configuration.

      • Description: Enter a description of the add-in.

Add-ins are unloaded as soon as a hook call is complete. To pass information between add-ins, use the SessionClass.Variable property.
