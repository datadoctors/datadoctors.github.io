Receipt Compiler and Renderer

Store Operations takes a receipt template string and iterates through loops, evaluation conditional clauses, and replaces variables with values. The result is a receipt string, in XML, that can be rendered.

For more information, see About Receipts.

Receipt Compiler

The following node types are processed by the receipt compiler. All of the other nodes are formatting nodes and are passed through to the renderer.

Node

Description

IF / THEN / ELSE

Conditional nodes

PROPERTIES

Comments; not processed

SET

Internal variables and public attributes

FOR / EACH / LOOP

Looping constructs

Variable replacements

Variable and constant evaluation

Receipt Renderer

The rendering process takes the result of the compile process and displays it in a window, prints it to an OPOS receipt printer, or renders it to a Windows printer. The following nodes are processed by the receipt renderer.

Node

Description

OPOS

OPOS specific functionality

MARGIN

Windows printer margin specifications

FONT

Font declarations

SUB

Subroutine definition

DOCUMENT

Document definition

CALL

Subroutine invocation

PICTURE

Graphics and images

BARCODE

Barcode display

TABLE

Table definitions

PARAGRAPH

Text-based paragraphs

TEXT

Generic text outputs

CURRENCY

Currency conversions
