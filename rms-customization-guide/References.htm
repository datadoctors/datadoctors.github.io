Reference

This section contains technical details about the programming elements used to customize Microsoft® Retail Management System.

  • QSRules Classes: Describes the design of the business logic tier of Store Operations POS. This section includes the documentation of critical classes, functions, and properties available for use by COM and HTML add-ins.

  • QSBridge Events: Describes the UI workflow tier of Store Operations POS. This tier contains events that can be raised by HTML add-ins to programmatically invoke the functionality available to the cashier through the POS UI.

QSBridge Events vs. QSRules Classes

QSBridge events and QSRules classes meet different customization needs. The following information will help you decide when to use each programming element.

  • Use QSBridge events in the following situations.

  • Your add-in needs to use the UI provided by RMS. QSRules does not interact with the dialog boxes used by the UI tier. For example, the handler for the RecallHold event displays a dialog box from which the user can select the transaction to recall, while the SessionClass.RecallTransaction function does not.

  • The functionality required by your add-in can be accomplished using QSBridge events. When an operation can be performed both by QSBridge events and QSRules classes, QSBridge events should usually be your first choice because they reside at a higher logical level. QSRules operates at a low level in the system and bypasses much of the workflow logic performed in the UI tier. For example, if you want to programmatically add an item to the current transaction, a QSBridge event is usually the better choice because the handler for the PerformAddItem event invokes the same logic called when the cashier adds a new item to the transaction. The handler automatically invokes the logic to prompt the cashier to enter a serial number or matrix item type. By contrast, if your add-in uses the Transaction.Entries collection to manually add a new entry to the transaction, the logic to prompt for a serial number or matrix item type is bypassed.

  • Use QSRules in the following situations.

  • Your add-in is a COM component. QSBridge events cannot be accessed through a COM component.

  • Your add-in requires information about the current transaction. Handlers for QSBridge events invoke UI workflow procedures, but do not have access to transaction information.

  • The functionality required by your add-in does not have an associated QSBridge event, or your add-in requires a finer level of control than is available in QSBridge events.
