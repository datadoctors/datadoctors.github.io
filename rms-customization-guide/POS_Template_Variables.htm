POS Template Variables

The following receipt template variables contain information about the current transaction.

 

Variable

Description

Cashier.Name

Cashier's full name.

Cashier.Firstname

Cashier’s first name.

Cashier.ID

ID of the entry in the Cashier table for the cashier.

Cashier.Loaded

Boolean value that indicates whether the cashier is loaded.

Cashier.Number

Number of the cashier.

Store.Name

Name of the store.

Store.Address1

First line of the address of the store.

Store.Address2

Second line of the address of the store.

Store.City

City of the address of the store.

Store.State

State code of the address of the store.

Store.Zip

Postal code of the address of the store.

Store.CityStateZip

City, state code, and postal code of the address of the store.

Store.ZipCity

Postal code and city of the address of the store.

Store.Country

Country code of the address of the store.

Store.Phone

Telephone number of the store.

Store.Fax

Facsimile number of the store.

Store.VATRregistrationNumber

Registration number for value-added tax systems.

Transaction.Balance

For internal use only.

Transaction.ChangeFoodStamp

Amount of change in food stamps to return to the customer.

Transaction.Date

Date on which the transaction occurred.

Transaction.Datetime

Date and time at which the transaction occurred.

Transaction.OrderBalance

Balance for the layaway, work order, quote, or back order at the time it was recalled.

Transaction.OrderBalanceNew

Current balance for the layaway, work order, quote or back order, including the present transaction.

Transaction.OrderID

ID in the order table for the layaway, work order, quote, or back order.

Transaction.OrderDueDate

Due date for the layaway, work order, quote, or back order.

Transaction.OrderDueTime

Due time for the layaway, work order, quote, or back order.

Transaction.Comment

Comment for the transaction.

Transaction.Deposit

Amount of the deposit applied to the current work order, quote, back order, or layaway.

Transaction.DepositPrevious

Amount of the deposit paid for the layaway, work order, quote, or back order at the time it was recalled.

Transaction.DepositTotal

Total deposit for the layaway, work order, quote, or back order. This includes both current and previous deposits.

Transaction.Discount

Amount of the discount for the transaction.

Transaction.DiscountMethod

Discount method used for the transaction. This is one of the following values:

  • discountmethodNone (0)

  • discountmethodDiscountFromRetail (1)

  • discountmethodDiscountFromCurrentPrice (2)

  • discountmethodMarkupFromCost (3)

  • discountmethodProfitMargin (4)

Transaction.DiscountPercent

Percentage of the discount for the transaction.

Transaction.DiscountPurchased

Amount of the discount for all items purchased in the transaction. In a sales transaction this is the total amount; in an order transaction this is the amount being picked up.

Transaction.DropAmount

Amount of the cash drop.

Transaction.DropComment

Comment for the cash drop.

Transaction.Name

Type of the transaction, for example "Sales".

Transaction.Number

TransactionNumber value of the entry in the Transaction table for the transaction. This variable to sales transactions and order pickups.

Transaction.Payment

Total payment for account receivables.

Transaction.PayoutAmount

Amount of the payout.

Transaction.PayoutComment

Comment for the payout.

Transaction.PayoutRecipient

Recipient for the payout.

Transaction.PriceLevel

Price level applied to the transaction. This is one of the following values:

  • pricelevelDefault (0)

  • priceLevelA (1)

  • priceLevelB (2)

  • priceLevelC (3)

Transaction.PurchaseLimitPurchase

Total for all items purchased in the transaction, for use in purchase limits. This amount includes sales tax for VAT tax systems.

Transaction.PurchaseLimitPoints

Total purchase limit points for the transaction. This is the sum of the purchase limit point values for each purchased item.

Transaction.PurchaseLimitUnits

Number of items purchased in the transaction, for use in purchase limits.

Transaction.Recalled

Boolean value that indicates whether the transaction was recalled.

Transaction.ReferenceID

ID of the entry in a table for the transaction. The table and field used depends on the type of the transaction as follows:

  • Sales: Transaction.TransactionNumber

  • Order: Order.ID

  • Drop/Payout: DropPayout.ID

  • Payment: Payment.ID

  • Other: NonTenderTransaction.ID

Transaction.ReferenceNumber

User-defined reference number for the transaction.

Transaction.SalesTax

Total sales tax for the transaction.

Transaction.SalesTaxPurchased

Total sales tax for all purchased and picked up items.

Transaction.SalesTaxOnOrder

Total sales tax for all ordered items.

Transaction.SubreferenceID

ID of the transaction in the OrderHistory table.

Transaction.SubTotal

Subtotal for the transaction.

Transaction.SubTotalPurchased

Subtotal for all purchased and picked up items.

Transaction.SubTotalOnOrder

Subtotal for all ordered items.

Transaction.Taxable

Boolean value that indicates whether tax can be applied to the transaction.

Transaction.Time

Time that the transaction was posted.

Transaction.Total

Total for the transaction, including tax.

Transaction.TotalDue

Total owed by the customer for deposits, payments, and items picked up, including tax.

Transaction.TotalPurchased

Total for all items purchased or picked up in the transaction, including tax.

Transaction.TotalExclsvPurchased

Total for all items purchased or picked up in the transaction, including tax.

Transaction.TotalOnOrder

Total for all ordered items in the transaction, including tax.

Transaction.TotalTendered

Total amount tendered for the transaction.

Transaction.TotalTenderIn

Total amount of tender taken in during the transaction.

Transaction.TotalTenderOut

Total amount of tender paid out as change for the transaction.

Transaction.TotalWeight

Sum of the weights of all weighed items in transaction.

Transaction.TotalWeightOnOrder

Sum of the weights of all weighed items on order.

Transaction.TotalWeightPurchased

Sum of the weights of all weighed items purchased or picked up.

Transaction.TotalQuantity

Total quantity of items in the transaction.

Transaction.TotalQuantityOnOrder

Total quantity of ordered items.

Transaction.TotalQuantityPurchased

Total quantity of all purchased and picked up items.

Transaction.EntryCount

Number of line items in the transaction.

Transaction.Type

Type of the transaction. This is one of the following values:

  • TransactionSales

  • TransactionWorkOrder

  • TransactionQuote

  • TransactionBackorder

  • TransactionLayaway

  • TransactionDrop

  • TransactionPayout

  • TransactionNoSale

  • TransactionAbortedTransaction

  • TransactionAccountPayment

  • TransactionXReport

  • TransactionZReport

  • TransactionZZReport

  • TransactionBlindCloseout

Transaction.PostingStatus

Not implemented.

Transaction.ReceiptTransactionName

Text value of the transaction type.

Transaction.ReceiptTransactionType

Obsolete; use ReceiptReferenceType.

Transaction.ReceiptTransactionNumber

ID of the entry in a table for the transaction. See ReferenceID for tables and values.

Transaction.ReceipttransacationNumberCaption

Caption for ReceiptTransactionNumber, for example, "Transaction #" or "Work Order #".

Transaction.ReceiptReferenceName

Name of the transaction receipt type, for example, "Sales Receipt" for sales and pickups.

Transaction.ReceiptReferenceType

Type of the transaction. This is one of the following values:

  • TransactionSales

  • TransactionWorkOrder

  • TransactionQuote

  • TransactionBackorder

  • TransactionLayaway

  • TransactionDrop

  • TransactionPayout

  • TransactionNoSale

  • TransactionAbortedTransaction

  • TransactionAccountPayment

  • TransactionXReport

  • TransactionZReport

  • TransactionZZReport

  • TransactionBlindCloseout

Transaction.ReceiptReferenceNumber

ID of the entry in the Order table if the transaction is an order; otherwise, zero.

Transaction.ReceiptReferenceNumberCaption

Caption for ReceiptReferenceNumber.

Transaction.ReceiptReturningItems

Boolean value that indicates whether any items in the transaction have a negative quantity.

Transaction.ReceiptReturnMode

Boolean value that indicates whether the transaction is in return mode (CTRL+F4).

Transaction.SalesTaxes.IsTransactionMember

Boolean value that indicates whether the sales tax is nonzero for the transaction.

Transaction.SalesTaxes.Description

Description of the sales tax.

Transaction.SalesTaxes.OnOrder

Total sales tax for ordered items.

Transaction.SalesTaxes.Purchased

Total sales tax for purchased and picked up items.

Transaction.SalesTaxes.Total

Total sales tax.

Transaction.SalesTaxes.ShowOnReceipt

Boolean value that indicates whether sales tax should be displayed on the receipt.
