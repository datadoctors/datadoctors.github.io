About the Retail Management System Databases

Both Retail Management System Store Operations and Retail Management System Headquarters use standard SQL databases to store all information. The databases are the following:

  • Store Operations Database: This database contains all information for a single store, including tables for transaction, item, and customer information.

  • Headquarters Database: This database is a superset of the Store Operations database; it contains most of the Store Operations tables as well as tables dedicated to Headquarters.

The following information applies to both databases.

  • Many tables have a trigger that is activated when a record in the table is deleted. The trigger stores information about the deletion in the RecordDeletedLog table.

  • The databases can be created, deleted, modified, and queried against by Store Operations Administrator and Headquarters Administrator.

  • Store Operations POS, Store Operations Manager, and Headquarters Manager open a connection to the databases when started and maintain the connection until exited.

  • When a column in one table is a reference to another table, the column name is the name of the referenced table followed by "ID". For example, the record in the TransactionEntry table with ItemID = 5 is a reference to the record in the Item table with ID = 5.
