Transaction Class

The Transaction class represents all types of POS transactions, including sales, work orders, back orders, quotes, and layaways. Only one Transaction object at a time exists under the main SessionClass object. The Transaction object contains the state of the entire transaction; this state is persisted to the database when the transaction is posted. Once posted, the Transaction object is cleared and reinitialized.

 

When To Use

The Transaction class is used to perform the following tasks.

  • Perform all sale operations at the POS.

  • Create back orders, work orders, and layaways.

  • Calculate totals, sales taxes, and discounts.

  • Persist transaction history to the database.

 

Procedures

The Transaction class defines the following procedures.

Method

Description

CalculateTotals

Calculates the subtotal, sales tax, and total for the transaction.

ClearCustomer

Clears the customer for the transaction.

ClearSalesRep

Clears the default sales representative for the transaction and for each entry in the transaction.

LoadSalesRep

Sets the default sales representative for the transaction to the representative with the specified ID.

MaximumFoodStampsAllowed

Determines the maximum amount of food stamps allowed for the transaction

Post

Posts the transaction to the database.

PostTenderValidate

Validates the transaction after the transaction is posted.

PreTenderValidate

Validates the transaction before the transaction is posted.

RevertToSavedPrices

Reverts the prices for each transaction entry to their saved values.

SavePrices

Saves the prices for each transaction entry.

SetCustomer

Sets the customer for the transaction.

SetDepositTotal

Sets the total deposit for the order.

SetDiscountPercentage

Sets the discount percentage for the transaction by the specified method.

SetDrop

Sets the cash drop amount and a comment for the transaction.

SetEntryDiscountReasonCodes

Sets the reason code used for discounts.

SetEntryReturnReasonCodes

Sets the reason code used for returns.

SetEntryTaxChangeReasonCodes

Sets the reason code to use for all tax changes.

SetPayout

Sets the payout amount, payout recipient, and a comment for the transaction.

SetPriceLevel

Sets the price level for the transaction.

SetTaxable

Sets a Boolean value that determines whether the transaction is taxable.

SetTransactionAmountOffOnce

Adjusts the discount on all entries in the transaction so that the total discount amount matches the specified value.

SetTransactionAmountOnce

Adjusts the price of all entries in the transaction so that the subtotal or total matches the specified amount.

SetTransactionPercentageOffOnce

Adjusts the discount on all entries in the transaction so that the total discount percentage matches the specified value.

SetTransactionType

Sets the transaction mode.

TendersForceValidation

Marks all tenders as validated.

TendersResetValidations

Marks all tenders as not validated.
