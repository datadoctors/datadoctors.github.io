Boolean Expression

A Boolean expression is any variable, relational expression, or Boolean expression that evaluates to either TRUE or FALSE.

Supported relational operators include the following. Note that the "<", ">", and "&" characters are reserved in XML; the escape sequences for these characters are shown in the table.

Operator

Meaning

=

Equal

&lt;

Less than (<)

&lt;=

Less than or equal to (<=)

&gt;

Greater than (>)

&gt;=

Greater than or equal to (>=)

&lt;&gt;

Not equal (<>)

&amp;

Logical AND (&)

|

Logical OR

The following are examples of Boolean expressions. ShowDiscounts is a user-defined Boolean variable; for more information about defining variables, see the SET element.

  • ShowDiscounts

  • Transaction.SubTotal &gt; 0

  • ShowDiscounts &amp; (Transaction.SubTotal &gt; 0)

  • (Transaction.Type = transactionSale) | (Transaction.Type = transactionPayment)

  • True

  • False

  • Yes

  • No

  • Len( string expression )

 

The Len() Function

The Len function of the receipt compiler returns a Boolean value that indicates whether the specified string expression is an empty string. The Len function returns TRUE if the string expression results in a string of length one or more and FALSE if it results in a string of length zero.

The receipt compiler does not support the full functionality of the Visual Basic Len function; the Boolean value TRUE is the equivalent of the numeric value -1, and FALSE is the equivalent of zero. For example, Len(MyStringVar), where MyStringVar is a previously defined string expression with the value "My String",  returns -1, not 9. Also, you cannot directly pass a string directly to the Len function.

The following XML example demonstrates how to use the Len function correctly. The example prints the text "Direct failed; Indirect succeeded." on the receipt.

<SET name="MyStringVar" custom="true" description="Test String" type="vbString">

    "My String"

</SET>

   

<DOCUMENT>

    <PARAGRAPH>

        <IF>

            <CONDITION> Len("My String") </CONDITION>

            <THEN>

                <TEXT> "Direct succeeded; " </TEXT>

            </THEN>

            <ELSE>

                <TEXT> "Direct failed; " </TEXT>

             </ELSE>

        </IF>

 

        <IF>

             <CONDITION> Len(MyStringVar) </CONDITION>

             <THEN>

                 <TEXT> "Indirect succeeded." </TEXT>

             </THEN>

             <ELSE>

                 <TEXT> "Indirect failed." </TEXT>

             </ELSE>

        </IF>

    </PARAGRAPH>

</DOCUMENT>
