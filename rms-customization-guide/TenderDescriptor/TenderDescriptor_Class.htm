TenderDescriptor Class

The TenderDescriptor class contains static information about a tender.

 

When To Use

The TenderDescriptor class is used to perform the following tasks.

  • View unchanging descriptive information about a tender through the Tender.Descriptor property.

  • Convert a currency value between the local currency on the computer and the default currency for the tender.

 

Procedures

The TenderDescriptor class defines the following procedures.

Method

Description

LocalAmount

Converts the specified amount of the default currency for the tender to its equivalent in local currency.

NativeAmount

Converts the specified amount of local currency to its equivalent in the default currency for the tender.

 

Properties

The TenderDescriptor class has the following properties.

 

AllowMultipleEntries

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the tender can be used on more than one line on the Tender dialog box.

 

CashBackFee

Data type: Currency

Access type: Read-only

Gets the fee applied when a debit card is used to get cash back.

 

CashBackLimit

Data type: Currency

Access type: Read-only

Gets the maximum amount that can be returned as cash back for a debit card.

 

CurrencyDescriptor

Data type: CurrencyDescriptor

Access type: Read-only

Gets the currency type for the tender.

 

DebitSurcharge

Data type: Currency

Access type: Read-only

Gets the surcharge applied for debit cards.

 

Denominations

Data type: TenderDenominations

Access type: Read-only

Gets the denominations for the tender.

 

Description

Data type: String

Access type: Read-only

Gets the description of the tender.

 

DoNotPopCashDrawer

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the cash drawer should be opened when the tender is used.

 

Key

Data type: String

Access type: Read-only

Gets the key of the tender descriptor.

 

MaximumAmount

Data type: Currency

Access type: Read-only

Gets the maximum amount of the tender that can be used for a transaction.

 

PreventOverTendering

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether the cashier can enter a higher value for the tender than the transaction total.

 

PrinterValidation

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether validation text is printed on the tender.

 

ScanCode

Data type: Integer

Access type: Read-only

Gets the keyboard macro code used to directly advance to the tender when a POS transaction is tendered.

 

SignatureRequired

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether a signature is required for the tender.

 

SupportCashBack

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether this tender can be used to receive cash back.

 

TenderType

Data type: TenderEnum

Access type: Read-only

Gets the type of the tender.

 

 

ValidationMask

Data type: String

Access type: Read-only

Gets the validation mask for a credit card tender.

 

ValidationText1

Data type: String

Access type: Read-only

Gets the first line of validation text to print on the tender.

 

ValidationText2

Data type: String

Access type: Read-only

Gets the second line of validation text to print on the tender.

 

ValidationText3

Data type: String

Access type: Read-only

Gets the third line of validation text to print on the tender.

 

VerifyViaEDC

Data type: Boolean

Access type: Read-only

Gets a Boolean value that indicates whether Electronic Draft Capture (EDC) is used to verify payment authorization.
