PrintReceipt Hook

The PrintReceipt hook is activated after a transaction is posted to the database and immediately before the receipt is printed.

HookType value

1

Parameter value

1

Because this hook is activated after the transaction is posted to the database, the Transaction.TransactionNumber property contains the number of the saved transaction.

The PrintReceipt hook is activated under the following conditions.

    • The cashier presses OK on the Tender window.

    1. The tender is validated.

    2. The pole display is updated to display the tender and change.

    3. The cash drawer is opened (if configured).

    4. The Post Transaction window is displayed with the change.

    5. The transaction is posted to the database.

    6. The PrintReceipt hook is activated.

    7. If the Process function of the add-in returns TRUE the receipt is printed; otherwise, an error occurs.

 

    • The cashier selects Print X Report, Print Z Report, Print ZZ Report, or Perform Blind Closeout from the Open/Close dialog box (F5 key).

    1. The Transaction.TransactionType property is set to the requested type if the terminal is online, the cashier has the right to perform the operation, and a transaction is not in progress.

    2. The requested report is generated. This operation populates the Report class and its subclasses in QSRules.

    3. The Post Transaction window is displayed.

    4. The transaction is posted to the database.

    5. The PrintReceipt hook is activated.

    6. If the Process function of the add-in returns TRUE the receipt is printed; otherwise, an error occurs.

 

    • The cashier performs a No Sale (F8 key).

    1. The Transaction.TransactionType property is set to transactiontypeNoSale if the terminal is online, the cashier has the right to perform the operation, and a transaction is not in progress.

    2. The user is prompted for a reason code (if configured).

    3. The cash drawer is opened (if configured).

    4. The Post Transaction window is displayed.

    5. The transaction is posted to the database.

    6. The PrintReceipt hook is activated.

    7. If the Process function of the add-in returns TRUE the receipt is printed; otherwise, an error occurs.

 

    • The cashier cancels a transaction (ESC Key).

    1. The Transaction.TransactionType property is set to transactiontypeAbortedTransaction if a transaction is in progress and the cashier has the right to perform the operation.

    2. The confirmation dialog box is displayed.

    3. An aborted transaction record is posted to the database.

    4. The PrintReceipt hook is activated.

    5. If the Process function of the add-in returns TRUE the receipt is printed; otherwise, an error occurs.
