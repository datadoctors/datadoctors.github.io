ROW Element

The ROW element defines a row in a table.

Attributes

None

Value

String expression. Use the "|" character to place succeeding text in the next column. The following examples show how to use the "|" character.

  • "column one | column 2"

column one

column 2

  • " | column2"

 

column2

  • Entry.Description "|" Entry.Price

Football

$9.99

Note that if you populate the table with variables, you must remember to place the "|" character in quotation marks.

Child Elements

None

Remarks

A table may contain any number of rows. The receipt compiler automatically generates a new page when the output exceeds the page bounds, as defined by margins and page header and footer. The table column headers are repeated on each page.

Do not start a table outside the area defined in the MARGIN element. This causes the header to be reprinted before each row that is not inside the margins. For example, assume the margins are defined as follows, where the variables are set as numeric values earlier in the receipt template.

<MARGIN>

    <COPIES> 1 </COPIES>
<TOP> MarginTop + HeaderHeight </TOP>
<BOTTOM> MarginBottom + FooterHeight </BOTTOM>
<LEFT> MarginLeft </LEFT>
<RIGHT> MarginRight </RIGHT>

</MARGIN>

A table is then positioned in the footer of the receipt template as follows, where the PaperHeight variable is set earlier in the receipt template.

<!-- Footer Table -->
<TABLE>

    <TOP> PaperHeight – MarginBottom – FooterHeight </TOP>
<!-- etc. -->

Notice that the top of the table begins in the exact space that has been reserved for the bottom margin. If the table has 3 rows, the expected result is the following:

HEADER

ROW

ROW

ROW

However, the receipt will actually print as the following:

HEADER

ROW

HEADER

ROW

HEADER

ROW

If you need to reserve extra space in addition to the bottom margin, such as for a footer in the preceding example, modify the value of the bottom margin to exclude the previously reserved space immediately before defining the footer table as follows:

<!-- Begin Footer -->

<!-- Reset the bottom margin to allow the table to be properly generated. -->
<MARGIN>
<BOTTOM> MarginBottom </BOTTOM>
</MARGIN>

<!-- Footer Table -->
<TABLE>
<TOP> PaperHeight – MarginBottom – FooterHeight </TOP>
<!-- etc. -->

Example

The following example displays a table that contains the description and price of each item in the transaction.

<TABLE>

    <BORDER>      tbAll        </BORDER>

    <PEN> <TEXT> 20 </TEXT> </PEN>

    <PENTB> <TEXT> 50 </TEXT> </PENTB>

    <PENLR> <TEXT> 100 </TEXT> </PENLR>

    <HEADERSHADE> vbGreen </HEADERSHADE>

 

    <COLUMNHEADER>

        <TEXT>      "Item"  </TEXT>

        <ALIGNMENT> "&lt;~" </ALIGNMENT>

        <WIDTH> PageWidth * 0.7 </WIDTH>

    </COLUMNHEADER>

    <COLUMNHEADER>

        <TEXT> "Price" </TEXT>

        <ALIGNMENT> "&lt;~" </ALIGNMENT>

        <WIDTH> PageWidth * 0.3 </WIDTH>

    </COLUMNHEADER>

 

    <FOR each="Entry">

        <ROW> Entry.Description "|" Entry.Price </ROW>

    </FOR>

</TABLE>
