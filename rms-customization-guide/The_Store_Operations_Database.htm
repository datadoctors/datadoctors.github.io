The Store Operations Database

The Store Operations database contains all information for a single store. This database contains the following types of tables.

  • Tables that define the store: These tables define the way the store operates, and are populated using Store Operations Manager. The Item, Department, Cashier, and Tax tables are examples of tables that define the store.

  • Tables that contain transaction records: These tables store transaction history, and are automatically populated by the POS. The Transaction and Order tables are examples of tables that contain transaction records.

Most tables in the Store Operations database have an ID column that is automatically generated by SQL Server. The ID column provides a unique number that is used to reference entries within the table. The Transaction and Batch tables are exceptions, in that the TransactionNumber column is the key field of the Transaction table and the BatchNumber column is the key field of the Batch table.

 

Transaction Table Population

During a POS transaction, all information entered by the cashier is stored in QSRules classes. This information is transferred to the database as follows:

  1. The cashier tenders the transaction.

  2. The POS copies the data in the QSRules classes to disconnected ADO Recordset objects.

  3. The Recordset objects are passed to the Gateway class.

  4. The Gateway class determines the transaction type and posts the data to the appropriate tables in the database.
