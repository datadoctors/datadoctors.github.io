Loyalty Add-in Hooks

The loyalty add-in uses hooks to invoke loyalty level functionality at key times during a transaction. The add-in uses hooks to invoke the following functionality:

  • Loyalty level calculation: The customer's loyalty level is calculated at the end of a transaction. For more information, see Loyalty Level Calculation in this topic.

  • Discount calculation: The loyalty discount is calculated whenever an item is added to a transaction, and at the end of a transaction. For more information, see Discount Calculation in this topic.

 

Loyalty Level Calculation

The customer's loyalty level is calculated by a class invoked from the PrintReceipt hook. This hook invokes the calculation after the transaction is tendered, but immediately before the receipt is printed. The class adds the final transaction subtotal to the customer's existing loyalty purchases. It then uses the updated purchase amount to determine whether the customer has reached a new loyalty level. Recalculating the loyalty level after the transaction has been tendered ensures that the discount for a new loyalty level will not be applied to a transaction that takes place in the previous level.

 

Discount Calculation

The discount for an item is calculated by a class invoked from the AddItem and TenderBegin hooks. Whenever an item is added to a transaction for which a loyalty customer is loaded, the customer's loyalty discount is compared to any other discounts that apply to the item. The greatest discount is applied to the item. The TenderBegin hook calculates the final discount after all items have been added, but immediately before the transaction is tendered.
