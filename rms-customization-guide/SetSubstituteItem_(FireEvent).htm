SetSubstituteItem Event

The handler for the SetSubstituteItem event replaces the current transaction entry with the specified item.

Handler Syntax

Public Function SetSubstituteItem( _

  ByVal SubstituteItemID As Long _

) As Boolean

Parameters

ID

Long value that contains the ID of the new item.

Return Values

Boolean value that indicates whether the operation was a success. The event handler returns TRUE if the operation was a success; otherwise, FALSE.

UI Equivalent Key

CTRL+F11, then select a substitute item

Remarks

The handler for the SetSubstituteItem event performs the following actions.

  1. Adds a new item (including any tag-along items) to the SessionClass.Transaction.Entries collection.

  2. Updates the transaction grid with the information for the new item (including any tag-along items).

  3. Recalculates totals, taxes, purchase limits, and discounts for the transaction.

  4. Reads the weight from the scale if the new item is a weighed item.

  5. Negates the quantity for the item if the transaction is in return mode.

  6. Sets the quantity for the item to one.

    • If the transaction is a sales transaction, sets TransactionEntry.QuantityPurchased to one.

    • If the transaction is an order transaction, sets TransactionEntry.QuantityOnOrder to one.

  1. Validates the quantities against the available quantities in the database.

  2. Prompts for a serial number for the item, if applicable.

  3. Prompts for price entry, if applicable.
