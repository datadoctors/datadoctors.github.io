PostTransaction Hook

The PostTransaction hook is activated immediately before a transaction is posted to the database.

HookType value

0

Parameter value

1

 

The PostTransaction hook is activated under the following conditions:

  • Sales Transactions. The cashier presses OK on the Tender window.

    1. The tender is validated.

    2. The pole display is updated to display the tender and change.

    3. The cash drawer is opened (if configured).

    4. The Post Transaction window is displayed showing the change amount.

    5. The PostTransaction hook is activated.

    6. If the Process function of the add-in returns TRUE, the transaction is posted to the database; otherwise, an error occurs.
       

    • Reports and Closeouts. The cashier selects Print X Report, Print Z Report, Print ZZ Report, or Perform Blind Closeout from the Open/Close dialog box (F5 key).

    1. The Transaction.TransactionType property is set to the requested type if the terminal is online, the cashier has the right to perform the operation, and a transaction is not in progress.

    2. The requested report is generated. This operation populates the Report class and its subclasses in QSRules.

    3. The Post Transaction window is displayed.

    4. The Batch table is updated.

    5. The PostTransaction hook is activated.

    6. If the Process function of the add-in returns TRUE, the transaction is posted to the database.

Note:  Because the Batch table is updated before the add-in is invoked, the Process function should not return False when running an X or Z report.

 

    • No Sales. The cashier performs a No Sale (F8 key). The Transaction.TransactionType property is set to transactiontypeNoSale if the terminal is online, the cashier has the right to perform the operation, and a transaction is not in progress.

    1. The user is prompted for a reason code (if configured).

    2. The cash drawer is opened (if configured).

    3. The Post Transaction window is displayed.

    4. The PostTransaction hook is activated.

    5. If the Process function of the add-in returns TRUE the transaction is posted to the database; otherwise, an error occurs.

 

  • Canceled Transactions. The cashier cancels a transaction (ESC Key).

    1. The Transaction.TransactionType property is set to transactiontypeAbortedTransaction if a transaction is in progress and the cashier has the right to perform the operation.

    2. The confirmation dialog box is displayed.

    3. The PostTransaction hook is activated.

    4. If the Process function of the add-in returns TRUE an aborted transaction record is posted to the database; otherwise, an error occurs.
