The Headquarters Database

The Headquarters database contains most of the Store Operations tables as well as tables dedicated to Headquarters. The Headquarters database contains the following types of tables.

  • Master tables: These tables are specific to Headquarters and contain information used to manage the business. Tables used to generate worksheets and tables that handle messaging are examples of Headquarters tables. Data in these tables is generally pushed down from Headquarters to the individual stores. Although this data may be modified at the stores, the changes can be overwritten the next time data is pushed down from Headquarters. Records in these tables are uniquely identified by the AutoID field. These tables also have an HQID field used to reference records.

  • Tables that contain transaction records: These tables are supersets of the Store Operations tables and contain store information for all stores. Data in these tables is pushed up from the individual stores to Headquarters. Records in these tables are uniquely identified by a combination of the StoreID and ID fields. For example, a record in the TransactionEntry table maps to a record in the Transaction table only if both the TransactionNumber and StoreID fields match. Although the StoreID field exists in the Store Operations database, the StoreID field in the Headquarters database is populated from the ID of the store communicating with Headquarters during upload of the data.

Global customers fit exactly into neither of these categories, since they may be created either at Headquarters or at the individual stores.

The direction in which a table is pushed can be determined by the identifying fields; tables that contain an HQID field are pushed down from Headquarters to the individual stores, while tables that contain a StoreID field are pushed up from the stores to Headquarters.
