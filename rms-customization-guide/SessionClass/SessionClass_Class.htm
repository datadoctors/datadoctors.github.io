SessionClass Class

The SessionClass class is the primary entry point for DLL and HTML add-ins. This object is instantiated once by the user interface workflow logic when Microsoft Retail Management System Store Operations POS starts, and is then passed to add-ins invoked by hooks, custom POS buttons, and QSBridge fire events. For more information, see About Add-Ins.

 

When To Use

The SessionClass class is used to perform the following tasks.

  • Access other objects through the properties of the SessionClass object created in the UI tier when the POS application is started.

  • Run a SQL command against the Microsoft Store Operations database using the OpenRecordset function.

  • Pass information between add-ins using the Variable property.

  • Access static transaction objects, such as SalesTaxDescriptors, CurrencyDescriptors, and Configuration. These objects are stored in the SessionClass object to eliminate the need to reinitialize them for every transaction.

 

Remarks

The most important classes accessed through the SessionClass object include the following:

  • Transaction: The head object in an object hierarchy that represents the current POS transaction.

  • Register: The head object in a hierarchy that represents the POS hardware.

  • Cashier:The object that represents the cashier currently logged into the system.

 

Procedures

The SessionClass class defines the following procedures.

Procedure

Description

HoldCurrentTransaction

Places the current transaction on hold and initializes a new transaction.

InitializeTransaction

Initializes a new transaction.

OpenRecordSet

Returns the Microsoft ActiveX Data Objects (ADO) record set retrieved by the specified SQL command.

PurgeMessages

Deletes all messages sent before the specified date.

RecallTransaction

Recalls the specified transaction.

SendMessage

Sends the specified message to the specified cashier.

TimeClockCorrect

Modifies the specified time clock entry.

TimeClockPunch

Punches the time clock for the specified cashier.
