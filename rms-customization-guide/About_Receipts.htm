About Receipts

Microsoft® Retail Management System receipts are highly customizable. Receipts are generated based on the transactions built up in various QSRules classes. The actual content and layout of the receipts are specified in XML receipt templates. These templates allow you to customize receipts using an XML design language; for more information, see Receipt Template Language.

Receipt templates provide:

  • Multiple receipt formats

  • Two receipt formats for each register to support multiple receipt printers

  • Different receipt templates for each transaction type

  • Preview option for each receipt template

  • Easy customization of receipt template properties

  • Import and export of receipt templates

The receipt templates are stored in the database as XML strings and can be imported from XML files and exported to XML files. The template files themselves can be edited with any XML editor.

The processing of the receipt templates is implemented in two passes: a compile pass and a rendering pass. The compile pass is always performed before the rendering pass. For more information, see Receipt Compiler and Renderer.

In addition to printing sales receipts, the following Store Operations features also use the following XML receipt templates:

  • Purchase Order

  • Purchase Order Receive

  • Inventory Transfer

  • Inventory Transfer Receive

  • Customer Account Statement

These are defined separately for each register using the Store Operations Administrator program.

Receipt templates can be grouped by the transaction type into a receipt format. Transaction types include:

  • Sales transaction

  • Cancelled transaction

  • Layaway transaction

  • Payment transaction

  • Pay out transaction

  • Cash drop transaction

  • Sales quote

  • Work order

  • X/Z/ZZ report

Each transaction type can use a different receipt template. Alternatively, one receipt template can accommodate multiple transaction types.

Each receipt format has a unique title and description. A register can reference two receipt formats (one for each printer) when printing receipts or saving receipts to a journal.

To see how the loyalty add-in modifies the default receipt template to print loyalty information, see Loyalty Receipt Sample.
