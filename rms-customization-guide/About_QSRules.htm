About QSRules

QSRules defines the Microsoft® Retail Management System business and database access logic associated with the POS. It is implemented as an ActiveX DLL (QSRules.dll) that is the primary API object layer used by add-ins to interact with RMS.

A reference to QSRules is passed to custom add-ins invoked from hooks, custom buttons, or one of the embedded browsers; this reference is used by add-ins to interact with the POS transaction in progress. The main QSRules class is SessionClass; all QSRules objects are accessed through the single SessionClass object instantiated when the POS application is started.

 

QSRules Object API

QSRules consists of a hierarchy of objects that reside under the head SessionClass object instantiated when the POS is started. The UI tier of the POS application uses many of the same QSRules objects available to custom add-ins. For example, the UI tier and custom add-ins both use the following code to access the last name of the customer.

sLName = Session.Transaction.Customer.LastName

QSRules also contains strongly-typed collection classes that can be enumerated. Collection class names are the pluralized version of the element classes they contain; for example, the Transaction.Entries property is a TransactionEntries object that contains an enumerable collection of TransactionEntry objects. The following example shows how to use an enumerable collection to access each element in the collection. For more information about the Process function, see The Process() Function.

Public Function Process(mySession As SessionClass) As Boolean

  

  Dim sItemList As String

  Dim Entry As TransactionEntry

  For Each Entry In mySession.Transaction.Entries

    sItemList = sItemList & Entry.Description & vbCrLf

  Next

  

  MsgBox sItemList

End Function
