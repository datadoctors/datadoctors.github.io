Microsoft Retail Management System

Microsoft® Retail Management System is a product suite that serves retailer needs all the way from a single store implementation up to a multi-store enterprise. Retail Management System consists of Store Operations and Headquarters. Store Operations provides point-of-sale and a back-office management capabilities at a single store location. Headquarters integrates two or more stores with a head office.

  • Store Operations consists of the following components.

Store Operations Administrator

Set configuration options for each sales register, such as online and offline database designation, path information, and receipt template locations.

Also, perform database management, such as SQL queries and database creation, deletion, backup, upgrade, and import.

Store Operations POS

Cashiers can ring up sales transactions, take orders, perform opening and closing functions, and other tasks.

POS runs at each sales register.

Store Operations Manager

Setup and manage registers, store inventory, taxes, tenders, reports, cashiers, customers, sales reps, and purchasing.

Manager runs on one computer, either a dedicated computer in the back office or one of the sales registers. Manager can be installed on more than one computer.

  • Headquarters consists of the following components.

Headquarters Administrator

Perform database management, such as SQL queries and database creation, deletion, backup, upgrade, and import. Also perform database configuration, including Headquarters database designation.

Administrator is usually installed on the same computer as Headquarters Manager.

Headquarters Manager

Manage all stores from a central location, including inventory, customers, taxes, tenders; and sales and inventory movement reports.

Headquarters Client

Provides the store-side link for communicating with the head office and processing the tasks created in Headquarters Manager.

Headquarters Server

Provides the head office-side link for communicating with the stores. It is responsible for sending tasks to the stores to be processed, as well as receiving data from the stores.

Communication Between Store Operations and Headquarters

Store Operations and Headquarters Client are installed at each store. Headquarters and Headquarters Server are installed at the head office.

Headquarters Server sends data from Headquarters to each of the stores, which is received and processed by each Headquarters Client. In return, Headquarters Server receives and processes data from the stores.
