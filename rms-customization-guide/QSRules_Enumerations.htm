QSRules Enumerations

QSRules uses enumerated types to provide human-readable options for procedures.

Enumerations

QSRules uses the following enumerated types.

Enumeration

Description

accountcolumnEnum

Defines the types that indicate whether an account in the accounting software integrated with RMS is a credit, debit, or both.

accountdueEnum

Defines the methods used to calculate the date a customer account is due.

accountinginterfaceEnum

Defines the types of accounting software that can be integrated with RMS.

cashierprivilegesEnum

Defines privileges that can be granted or denied to cashiers.

configurationoptionEnum

Lists the configuration options in the Manager application.

costupdatemethodEnum

Defines the methods used to update item costs when inventory is received.

discountmethodEnum

Defines the methods used to calculate the discount.

LimitTimeFrameEnum

Defines the available purchase limit enforcement periods.

LimitTypeEnum

Defines the available types of purchase limit.

overshortlimitEnum

Defines the different methods that can be used to limit the over/short error amount allowed for a cashier.

PriceCalculationRuleEnum

Defines the rules used to calculate the price of an item when it is added to a transaction.

pricelevelEnum

Defines the available price levels.

pricesourceEnum

Defines the possible sources of the price.

recallEnum

Defines the types of transaction recall.

StatusCategoryEnum

Contains values that indicate the general status of the most recent operation.

StatusCodeEnum

Contains values that indicate the status of the most recent operation.

TaxBasisEnum

Defines the methods that can be used to calculate tax.

taxfieldEnum

Defines the fields used to calculate tax on a per-customer basis.

TaxSystemEnum

Defines the tax systems used by RMS.

transactiontypeEnum

Defines the types of transaction.
