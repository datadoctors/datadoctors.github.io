SET Element

The SET element defines a variable for use in the receipt template.

Attributes

Attribute

Value

Description

name

String expression

Required. Name of the variable. The name is used to reference the variable in the receipt template.

custom

Boolean expression

Optional. Indicates whether the value of the variable can be modified in the Receipt Templates dialog box in Manager. The default value is FALSE.

description

String expression

Optional. Description of the variable. If the custom attribute is TRUE, the description is used to describe the variable in the Receipt Templates dialog box in Manager. If this value is not specified, the name of the variable appears on the dialog box.

type

  • VbLong

  • VbDouble

  • VbString

  • VbCurrency

  • VbCurrencyWords

  • VbDateTime

  • VbBoolean

Required. Visual Basic type of the variable.

Value

Value of the variable. The accepted values are dependent on the type attribute. Numeric, Boolean, and Currency variables are formatted according to the local regional settings when displayed on the receipt.

Child Elements

None

Remarks

This element declares and sets a variable for the receipt template. The variable can then be used in an expression as if it were predefined receipt template variable. The value of a variable can be changed during receipt processing by adding a SET element with the same name attribute.

Example

The following example shows how to declare, set, and use variables in a receipt template. This example centers the text in the footer variables on the receipt. While these lines of text could be printed on the receipt without using variables, the advantage of using variables is that the value of variables with a custom attribute of TRUE can be modified through the Receipt Templates dialog box in Manager. This allows the receipt template to be modified without editing the raw XML.

<SET name="FooterLine1" custom="true" description="Footer Line 1" type="vbString">  

    "Thank you for shopping"

</SET>

<SET name="FooterLine2" custom="true" description="Footer Line 2" type="vbString">   

    Store.Name    

</SET>

<SET name="FooterLine3" custom="true" description="Footer Line 3" type="vbString">  

    "We hope you'll come back soon!"

</SET>

 

<DOCUMENT>

    <TABLE>

        <BORDER> tbNone   </BORDER>

        <COLUMNHEADER>

            <ALIGNMENT> "^~"      </ALIGNMENT>

            <WIDTH>     PageWidth </WIDTH>

            <IF>

                <CONDITION> Len(FooterLine1) </CONDITION>

                <THEN>

                    <TEXT> FooterLine1 </TEXT>

                </THEN>

            </IF>

        </COLUMNHEADER>

 

        <IF>

            <CONDITION> Len(FooterLine2) </CONDITION>

            <THEN>

                <ROW> FooterLine2 </ROW>

            </THEN>

        </IF>

        <IF>

            <CONDITION> Len(FooterLine3) </CONDITION>

            <THEN>

                <ROW> FooterLine3 </ROW>

            </THEN>

        </IF>

    </TABLE>

</DOCUMENT>
