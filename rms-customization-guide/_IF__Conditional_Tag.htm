IF Element

The IF element determines which content is rendered on the receipt.

Attributes

None

Value

None

Child Elements

Element

Content

Description

CONDITION

Boolean expression

Required. If the condition evaluates to TRUE, the content of the THEN element is evaluated; otherwise, the content of the ELSE element is evaluated.

THEN

Any content

Required. Content to render if the condition is true.

ELSE

Any content

Optional. Content to render if the condition is false.

Remarks

The IF element is used to render specific content only if a preexisting condition is met. Alternate content can be rendered if the condition is not met. The IF element can appear anywhere in the receipt template. It can be used to alter values in other objects as well as include or exclude whole sections.

IF elements can be nested to create a more complex condition. For details, see the example below.

Certain characters are reserved in XML. You must use the escape sequences for these characters to use them in the CONDITION element. The escape sequences are the following:

    • "<" = "&lt;"

    • ">" = "&gt;"

    • "&" = "&amp;"

Example

The following example shows how to create a nested conditional structure. The example prints the text "Thank you." on the receipt if the customer spends less than $100.00, prints "Thank you for spending over $100!" if the customer spends between $100.00 and $199.99, and prints "Thank you for spending over $200!" if the customer spends $200.00 or more.

<IF>

   <CONDITION> Transaction.Total &lt; 100.00 </CONDITION>

   <THEN>

      <TEXT> "Thank you." </TEXT>

   </THEN>

   <ELSE>

      <IF>

         <CONDITION>

            (Transaction.Total &gt;= 100.00) &amp; (Transaction.Total &lt; 200.00)   

         </CONDITION>

         <THEN>

            <TEXT> "Thank you for spending over $100!" </TEXT>

         </THEN>

         <ELSE>

            <TEXT> "Thank you for spending over $200!" </TEXT>

         </ELSE>

      </IF>

   </ELSE>

</IF>
