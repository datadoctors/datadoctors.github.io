Tailspin Toys Scenario

The customization kit was created to help developers new to RMS learn to customize Manager and POS. The customization kit shows how to customize RMS by presenting and solving a sample business scenario. In the scenario, the owner of Tailspin Toys wants to reward customers for shopping at his store. The customization kit shows how this might be accomplished by creating customizations that implement a loyalty program. This section highlights the elements of the loyalty program solution that specifically relate to RMS customization. The term "loyalty add-in" is used in this section to indicate the following loyalty program customizations:

  • Loyalty program custom button; for more information, see Loyalty Add-in Custom Buttons

  • Hooks used by the loyalty program add-in; for more information, see Loyalty Add-in Hooks

  • COM components written in Visual Basic .NET that are called from hooks to apply loyalty discounts to items and compute loyalty levels

  • HTML scripts used by the loyalty program add-in; for more information, see Loyalty Add-in Browser, Loyalty Add-in Status Display, and Loyalty Add-in Net Display

  • Loyalty program Active reports; for more information, see Loyalty Add-in Reports

  • Loyalty program XML receipt template; for more information, see Loyalty Add-in Receipt Templates

The loyalty add-in also uses the following elements:

  • Wizards that automate deployment of the add-in; for more information, see Loyalty Add-in Installation

  • SQL database that stores the loyalty program information; for more information, see Loyalty Add-in Database

  • Encryption of the connection string used to connect to the loyalty database; for more information, see Loyalty Add-in Security
